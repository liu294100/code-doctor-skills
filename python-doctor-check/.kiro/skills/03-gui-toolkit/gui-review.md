# Python GUI 小工具代码审查 Skill

当用户打开或提交 GUI 项目文件时，按照以下维度进行全面审查。

---

## 一、架构审查

### 1.1 UI 与逻辑分离
- 视图代码中是否混入了业务逻辑（文件操作、网络请求、数据处理）
- 业务逻辑是否可以脱离 UI 框架独立测试
- 是否存在"上帝类"（一个类包含所有 UI 和逻辑）
- 数据模型是否与 UI 控件解耦

### 1.2 模块组织
- 是否按功能划分模块（views / services / models / widgets）
- 自定义控件是否可复用
- 是否存在循环导入
- 入口文件是否简洁（只负责启动）

---

## 二、线程安全审查

### 2.1 主线程阻塞
- 是否在主线程执行耗时操作（文件读写、网络请求、大量计算）
- 耗时操作是否放到后台线程/进程
- UI 是否会因为阻塞操作而卡顿无响应

### 2.2 UI 更新安全
- 是否在子线程直接操作 UI 控件（严重错误，会导致崩溃）
- Tkinter：是否使用 `root.after()` 或 `queue.Queue` 回调主线程
- PyQt：是否使用信号（Signal）机制更新 UI
- 是否正确处理线程间数据传递

### 2.3 资源竞争
- 共享数据是否使用锁保护
- 文件操作是否有并发冲突风险
- 后台任务取消机制是否正确（避免僵尸线程）

---

## 三、用户体验审查

### 3.1 交互反馈
- 耗时操作是否有进度指示（进度条、加载动画、状态栏提示）
- 操作结果是否有明确反馈（成功/失败提示）
- 按钮在操作进行中是否禁用（防止重复点击）
- 是否有取消操作的途径

### 3.2 错误处理
- 是否有全局异常捕获（防止未处理异常导致崩溃）
- 错误提示是否对用户友好（非技术性堆栈信息）
- 输入校验是否及时（实时校验 vs 提交时校验）
- 文件不存在、权限不足等常见错误是否处理

### 3.3 窗口与布局
- 窗口是否设置了合理的默认大小和最小尺寸
- 布局是否使用布局管理器（而非绝对坐标）
- 窗口缩放时控件是否正确自适应
- 高 DPI 屏幕是否显示正常

### 3.4 可访问性
- 控件是否有文字标签或 tooltip
- 是否支持键盘操作（Tab 切换、Enter 确认、Esc 取消）
- 字体大小是否可调或足够大
- 颜色对比度是否足够

---

## 四、跨平台审查

### 4.1 路径处理
- 文件路径是否使用 `pathlib.Path` 而非字符串拼接
- 是否硬编码了平台特定路径分隔符（`\` vs `/`）
- 用户数据目录是否使用 `platformdirs` 获取
- 临时文件是否使用 `tempfile` 模块

### 4.2 平台差异
- 字体是否在不同平台可用（使用系统默认字体或内嵌字体）
- 快捷键是否适配平台（Ctrl vs Cmd）
- 文件对话框是否使用原生对话框
- 系统托盘功能是否跨平台兼容

### 4.3 编码问题
- 文件读写是否指定了编码（`encoding="utf-8"`）
- 是否处理了不同平台的换行符差异
- 中文路径是否能正确处理

---

## 五、性能审查

### 5.1 启动性能
- 启动时是否加载了不必要的资源
- 是否使用延迟加载（按需导入模块）
- 初始化是否可以异步执行

### 5.2 内存管理
- 大文件是否使用流式处理
- 图片资源是否及时释放
- 是否存在内存泄漏（事件监听器未移除、循环引用）
- 列表/表格大数据量是否使用虚拟化

### 5.3 渲染性能
- 是否存在不必要的 UI 重绘
- 大量数据更新是否使用批量更新
- 动画是否流畅（帧率是否合理）

---

## 六、安全审查

### 6.1 输入安全
- 用户输入是否做了校验和清洗
- 文件路径是否防止路径穿越
- 是否存在命令注入风险（`os.system`、`subprocess` 拼接用户输入）
- 反序列化是否安全（不要 `pickle.loads` 不可信数据）

### 6.2 数据安全
- 敏感数据（密码、API Key）是否加密存储
- 配置文件权限是否合理
- 日志中是否打印了敏感信息

---

## 七、审查输出格式

```
【问题分类】架构 / 线程安全 / 用户体验 / 跨平台 / 性能 / 安全
【严重程度】🔴 严重（必须修复）/ 🟡 警告（建议修复）/ 🔵 建议（可优化）
【问题位置】文件名:类名.方法名 (行号)
【问题描述】具体描述问题是什么
【风险说明】不修复可能导致什么后果
【修复建议】具体的修复方案和示例代码
```

审查完成后，给出整体评估：
1. 代码质量评分（1-10 分）
2. 最需要优先修复的 Top 3 问题
3. 用户体验改进建议
