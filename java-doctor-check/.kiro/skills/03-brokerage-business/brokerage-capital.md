# 互联网券商资金系统专项 Skill（含保证金融资账户）

参考富途/moomoo 等标杆产品，针对互联网券商多币种资金体系、保证金账户、融资融券的深度审查与设计指导。

## 一、资金账户体系架构

### 1.1 账户类型与层级

```
客户（Client）
├── 现金账户（Cash Account）
│   ├── HKD 子账户
│   ├── USD 子账户
│   └── CNY 子账户
├── 保证金账户（Margin Account）
│   ├── HKD 子账户
│   ├── USD 子账户
│   └── CNY 子账户
├── 融资账户（Financing Account）— 港股融资打新专用
│   └── HKD 子账户
└── 期货账户（Futures Account）
    ├── HKD 子账户
    └── USD 子账户
```

### 1.2 现金账户模型

```java
class CashAccount {
    String accountId;
    String currency;          // 币种
    BigDecimal totalAssets;   // 总资产 = cash + marketValue
    BigDecimal cash;          // 现金余额
    BigDecimal frozenCash;    // 冻结资金（委托冻结 + 提现冻结）
    BigDecimal availableCash; // 可用资金 = cash - frozenCash
    BigDecimal marketValue;   // 持仓市值
    BigDecimal inTransit;     // 在途资金（T+2 未交收）
    BigDecimal todayPnL;      // 今日盈亏
    BigDecimal totalPnL;      // 累计盈亏
}
```

**现金账户规则**
- 买入：可用资金 >= 委托金额 + 预估手续费
- 卖出：可用股份 >= 委托数量
- 不能透支：可用资金不能为负
- 交收周期：港股 T+2，美股 T+1，A 股 T+1
- 在途资金：卖出后资金在交收完成前为"在途"状态，不可提取但可用于买入

### 1.3 保证金账户模型（核心）

```java
class MarginAccount {
    String accountId;
    String currency;

    // === 资产端 ===
    BigDecimal cash;              // 现金
    BigDecimal longMarketValue;   // 多头持仓市值
    BigDecimal shortMarketValue;  // 空头持仓市值（融券）
    BigDecimal totalAssets;       // 总资产 = cash + longMV

    // === 负债端 ===
    BigDecimal marginLoan;        // 融资借款本金
    BigDecimal accruedInterest;   // 应计利息
    BigDecimal shortBorrowValue;  // 融券借入市值

    // === 净值 ===
    BigDecimal netLiquidation;    // 净清算价值 = totalAssets - marginLoan - accruedInterest
    BigDecimal excessLiquidity;   // 超额流动性 = netLiquidation - maintenanceMargin

    // === 保证金 ===
    BigDecimal initialMargin;     // 初始保证金要求
    BigDecimal maintenanceMargin; // 维持保证金要求
    BigDecimal availableMargin;   // 可用保证金 = netLiquidation - initialMargin

    // === 购买力 ===
    BigDecimal buyingPower;       // 购买力 = availableMargin / initialMarginRate
    BigDecimal overnightBP;       // 隔夜购买力（通常 2:1 杠杆）
    BigDecimal intradayBP;        // 日内购买力（通常 4:1 杠杆，美股 PDT）

    // === 风控指标 ===
    BigDecimal marginRatio;       // 保证金比例 = netLiquidation / totalAssets
    BigDecimal riskLevel;         // 风险等级：安全/警告/危险/强平
}
```

### 1.4 保证金计算引擎

**初始保证金（Initial Margin）— 开仓时要求**
```
股票多头初始保证金 = 持仓市值 × 初始保证金比例
  港股：通常 25%-50%（根据股票流动性和波动性分级）
  美股：Reg T 要求 50%
  期权：根据策略不同计算方式不同

初始保证金比例分级（港股示例）：
  蓝筹股（恒指成分股）：25%
  大型股（市值 > 100亿）：30%
  中型股（市值 10-100亿）：50%
  小型股/创业板：70%-100%（部分不可融资）
```

**维持保证金（Maintenance Margin）— 持仓期间要求**
```
维持保证金 = Σ(每只股票市值 × 该股票维持保证金比例)
  通常为初始保证金的 70%-80%
  港股：通常 15%-40%
  美股：通常 25%（FINRA 最低要求）
```

**保证金比例实时计算**
```
保证金比例 = 净清算价值 / 总持仓市值 × 100%

风险等级判定：
  安全（Safe）：保证金比例 > 初始保证金比例
  警告（Warning）：维持保证金比例 < 保证金比例 <= 初始保证金比例
  危险（Danger）：强平线 < 保证金比例 <= 维持保证金比例
  强平（Force Liquidation）：保证金比例 <= 强平线
```

**购买力计算**
```
可用保证金 = 净清算价值 - 初始保证金要求
购买力 = 可用保证金 / 目标股票的初始保证金比例

示例：
  净清算价值 = 100万
  当前初始保证金要求 = 40万
  可用保证金 = 60万
  目标股票初始保证金比例 = 25%
  购买力 = 60万 / 25% = 240万
```

## 二、融资业务（Margin Financing）

### 2.1 融资业务流程

```
客户开通保证金账户 → 签署融资协议 → 设定融资额度
                                      ↓
买入股票时自动融资：
  可用资金不足 → 自动使用融资额度 → 产生融资借款
  融资金额 = 买入金额 - 可用现金（如果可用现金不足）
                                      ↓
持仓期间：
  每日计息 → 利息累计 → 月结扣息
  实时监控保证金比例 → 低于警戒线通知 → 低于强平线强制平仓
                                      ↓
还款方式：
  卖出股票 → 卖出所得优先偿还融资借款
  主动还款 → 入金后手动偿还
  自动还款 → 入金时自动偿还（可配置）
```

### 2.2 融资额度管理

**额度层级**
```
总融资额度（公司级）
  └── 单客户融资额度
       ├── 港股融资额度
       ├── 美股融资额度
       └── 单只股票融资额度上限
```

**额度计算**
```
客户融资额度 = min(
    客户信用评级对应的最大额度,
    客户净资产 × 最大杠杆倍数,
    公司剩余可融资额度
)

单只股票可融资金额 = min(
    该股票市值 × (1 - 初始保证金比例),
    该股票集中度上限,
    客户剩余融资额度
)
```

**融资标的管理**
- 可融资标的名单：根据股票流动性、波动性、市值等因素确定
- 保证金比例分级：不同股票不同比例，定期调整
- 集中度限制：单只股票融资不超过客户总融资的 X%
- 黑名单：高风险股票（如即将退市、被做空）不可融资
- 名单更新：每日/每周更新，变更时通知受影响客户

### 2.3 利息计算

**计息规则**
```
日利息 = 融资余额 × 年利率 / 365（或 360，取决于币种）
  港币融资：通常 P + X%（P = 最优惠利率）或固定利率 6%-8%
  美元融资：通常 SOFR + X% 或固定利率 5%-7%

计息方式：
  按日计息，按月结息
  T 日借款，T 日开始计息
  T 日还款，T 日停止计息（还款当日不计息）

利息结算：
  每月最后一个交易日结息
  利息从现金余额中扣除
  现金不足时，利息转为融资借款（利滚利）
```

**利息展示**
- 实时展示：当前融资余额、日利息、累计未结利息
- 历史查询：每月利息账单、利率变更记录
- 利率通知：利率调整时提前通知客户

### 2.4 强制平仓（Force Liquidation）

**触发条件**
```
保证金比例 <= 强平线（通常 15%-20%）
或
追加保证金通知（Margin Call）后 T+2 日未补足
```

**强平流程**
```
保证金比例实时监控
  → 低于警戒线（如 30%）：发送预警通知（App推送 + 短信 + 邮件）
  → 低于追保线（如 25%）：发送 Margin Call 通知，要求 T+2 内补足
  → 低于强平线（如 15%）：触发强制平仓
     → 计算需要平仓的金额
     → 选择平仓标的（优先平仓保证金比例最高的股票）
     → 生成强平委托（市价单）
     → 平仓直到保证金比例恢复到安全线以上
     → 通知客户平仓结果
```

**强平策略**
- 平仓金额 = (维持保证金 - 净清算价值) / (1 - 维持保证金比例)
- 平仓标的选择优先级：
  1. 保证金比例最高的股票（释放最多保证金）
  2. 流动性最好的股票（容易成交）
  3. 亏损最大的股票（止损）
- 平仓限制：
  - 单次平仓不超过需要的金额（不过度平仓）
  - 碎股处理：平仓可能产生碎股
  - 停牌股票：跳过，等复牌后处理

## 三、多币种资金管理

### 3.1 多币种架构

```
客户总资产（以基础币种展示，如 HKD）
├── HKD 子账户
│   ├── 现金：100,000 HKD
│   ├── 港股持仓市值：500,000 HKD
│   └── 融资借款：-200,000 HKD
├── USD 子账户
│   ├── 现金：10,000 USD
│   ├── 美股持仓市值：50,000 USD
│   └── 融资借款：-20,000 USD
└── CNY 子账户
    ├── 现金：50,000 CNY
    └── A股持仓市值：200,000 CNY

总资产（HKD）= HKD资产 + USD资产×汇率 + CNY资产×汇率
```

### 3.2 换汇系统

**换汇场景**
- 主动换汇：客户手动将一种币种换为另一种
- 自动换汇：买入美股时 HKD 不足，自动将 HKD 换为 USD
- 结算换汇：港股通结算时按结算汇率换汇

**换汇设计**
```
换汇请求 → 汇率查询（实时汇率 + 点差）
         → 金额计算（扣除手续费）
         → 资金校验（源币种余额充足）
         → 执行换汇（原子操作：扣源币种 + 加目标币种）
         → 换汇记录
```

**汇率管理**
- 汇率来源：银行间市场汇率 / 合作银行报价
- 汇率更新频率：交易时段每 5-15 秒更新
- 买入价/卖出价：中间价 ± 点差
- 点差策略：大额换汇点差更小（阶梯定价）
- 汇率锁定：下单时锁定汇率，有效期 30 秒

### 3.3 出入金管理

**入金渠道**
```
银行转账（电汇）：
  客户银行 → 券商银行账户 → 入金审核 → 到账
  到账时间：同行即时，跨行 1-2 个工作日
  审核要点：姓名一致性、来源合规性

eDDA（电子直接扣款授权，香港）：
  绑定银行账户 → 发起扣款 → 银行确认 → 到账
  到账时间：即时或 T+1

FPS（转数快，香港）：
  即时到账，7×24 小时
  限额：每日限额由银行设定

ACH（美国）：
  绑定银行账户 → 发起转账 → 3-5 个工作日到账

Wire Transfer（电汇，美国）：
  即时发起 → 1-2 个工作日到账
```

**出金规则**
- 可提取金额 = 可用资金 - 在途资金 - 未交收资金 - 最低保留金额
- 保证金账户：提取后保证金比例不能低于初始保证金要求
- 出金审核：大额出金（如 > 50 万）需人工审核
- 出金频率限制：每日出金次数限制
- 出金到账：原路返回（入金渠道）
- 反洗钱检查：出金前进行 AML 检查

## 四、资金清算与交收

### 4.1 交收周期

| 市场 | 交收周期 | 说明 |
|------|---------|------|
| 港股 | T+2 | CCASS 交收 |
| 美股 | T+1 | DTCC 交收（2024年起从T+2改为T+1） |
| A股（北向） | T+1 | 通过 HKSCC-ChinaClear |
| 期货 | T+0 | 逐日盯市，每日结算 |
| 期权 | T+1 | 行权后 T+1 交收 |

### 4.2 日终资金清算流程

```
收市后 → 接收交易所清算文件
       → 成交核对（本地 vs 交易所，100% 一致）
       → 资金清算
         ├→ 买入清算：确认冻结资金 → 转为应付交收款
         ├→ 卖出清算：计算应收交收款 → 更新在途资金
         ├→ 手续费清算：佣金 + 交易征费 + 印花税 + 交收费
         ├→ 融资利息清算：计算当日利息
         └→ 公司行为清算：分红/送股/拆合股
       → 交收处理
         ├→ T+2 到期的应收款 → 转为可用资金
         └→ T+2 到期的应付款 → 从冻结资金中扣除
       → 保证金重算：按收盘价重新计算所有保证金账户
       → 报表生成：客户对账单、资金变动明细
       → 初始化下一交易日
```

### 4.3 手续费体系

**港股手续费**
```
佣金 = max(成交金额 × 佣金费率, 最低佣金)
  互联网券商通常：万三佣金，最低 3 港元
平台使用费 = 每笔 15 港元（部分券商收取）
交易征费 = 成交金额 × 0.00278%
交易费 = 成交金额 × 0.00565%
印花税 = max(成交金额 × 0.13%, 1 港元)（取整到元）
交收费 = 成交金额 × 0.002%（最低 2 港元，最高 100 港元）
```

**美股手续费**
```
佣金 = 每股 $0.0049（或每笔 $0.99，取决于定价模式）
平台费 = 每笔 $1（部分券商）
SEC Fee = 卖出金额 × $8 / 1,000,000（仅卖出）
TAF Fee = 卖出股数 × $0.000166（仅卖出，最高 $5.95）
```

**费率配置化设计**
- 费率表：按市场 × 费用类型 × 客户等级配置
- 费率生效时间：支持预设未来生效的费率
- 费率计算精度：中间过程保留 6 位小数，最终结果按币种精度取整
- 费率审计：费率变更需审批，保留变更历史

## 五、IPO 融资认购

### 5.1 IPO 融资模型

```
IPO 融资认购：
  客户自有资金（保证金）：认购金额 × 保证金比例（通常 5%-10%）
  券商融资部分：认购金额 × (1 - 保证金比例)
  融资利息：融资金额 × 年利率 / 365 × 融资天数

示例：
  认购金额：100,000 HKD
  保证金比例：10%
  客户出资：10,000 HKD
  券商融资：90,000 HKD
  融资利率：年化 3.98%
  融资天数：7 天（认购期 + 公布结果）
  融资利息：90,000 × 3.98% / 365 × 7 = 68.88 HKD
```

### 5.2 IPO 融资系统设计

**额度管理**
```
公司 IPO 融资总额度
  └── 单只 IPO 融资额度
       └── 单客户 IPO 融资额度 = min(
             客户信用额度,
             单只 IPO 最大融资额度,
             剩余可分配额度
           )
```

**认购流程**
```
IPO 开放认购 → 客户选择认购档位和融资倍数
             → 校验：保证金充足 + 融资额度充足
             → 冻结保证金 + 预占融资额度
             → 认购截止 → 汇总提交
             → 等待中签结果
               ├→ 中签：
               │   扣除保证金 + 实际融资 = 中签金额
               │   融资部分开始计息
               │   股份入账（上市日）
               │   客户可选择：持有（自动还款）或上市首日卖出
               └→ 未中签：
                   解冻保证金
                   释放融资额度
                   退还多余资金（如有）
```

**风控要点**
- 单客户单只 IPO 最大融资倍数限制（如最高 10 倍）
- 公司总 IPO 融资敞口控制
- 暗盘交易风控：IPO 融资客户暗盘卖出时优先偿还融资
- 上市首日破发风险：融资客户亏损可能超过保证金，需要追保机制

## 六、资金安全与对账

### 6.1 资金安全架构

**三层对账体系**
```
第一层：实时对账（每笔交易后）
  每笔资金变动后校验：借方合计 = 贷方合计
  异常立即告警，暂停相关操作

第二层：准实时对账（每 5 分钟）
  汇总所有账户余额 vs 总账余额
  差异超过阈值告警

第三层：日终对账（T+1）
  内部账本 vs 银行流水
  内部账本 vs 交易所清算文件
  内部账本 vs 托管行报表
  差异逐笔核对，自动/人工处理
```

**资金隔离**
- 客户资金与公司自有资金严格隔离
- 客户资金存放在独立的信托账户（Trust Account）
- 不同币种资金分别存放
- 融资资金来源与客户资金分开管理

### 6.2 记账模型

**复式记账**
```
每笔资金变动必须同时记录借方和贷方：

买入股票：
  借：证券持仓（资产增加）
  贷：银行存款（资产减少）
  贷：应付佣金（负债增加）

融资买入：
  借：证券持仓（资产增加）
  贷：银行存款（自有资金部分）
  贷：融资借款（负债增加）

卖出股票：
  借：银行存款（资产增加）
  借/贷：投资收益（盈亏）
  贷：证券持仓（资产减少）
```

**流水设计**
```java
class FundFlow {
    String flowId;           // 流水号（全局唯一）
    String accountId;        // 账户ID
    String currency;         // 币种
    String bizType;          // 业务类型：TRADE/FEE/INTEREST/DEPOSIT/WITHDRAW/EXCHANGE/DIVIDEND/IPO
    String direction;        // 方向：IN/OUT/FREEZE/UNFREEZE
    BigDecimal amount;       // 金额
    BigDecimal balanceBefore;// 变动前余额
    BigDecimal balanceAfter; // 变动后余额
    String refId;            // 关联业务ID（委托号/入金单号等）
    Long createTime;         // 创建时间
    String remark;           // 备注
}
// 流水表只 INSERT，禁止 UPDATE/DELETE
```

## 七、资金系统审查检查清单

### 资金安全（P0）
- [ ] 所有资金变动是否有流水记录
- [ ] 流水表是否只追加不修改
- [ ] 余额变更是否有并发控制（SELECT FOR UPDATE / 乐观锁）
- [ ] 金额计算是否全部使用 BigDecimal
- [ ] 是否存在资金操作不在事务中的情况
- [ ] 对账机制是否覆盖所有资金链路
- [ ] 客户资金是否与公司资金隔离

### 保证金计算（P0）
- [ ] 保证金比例计算是否实时准确
- [ ] 购买力计算是否考虑了所有持仓和负债
- [ ] 强平触发条件是否正确
- [ ] 强平金额计算是否合理（不过度平仓）
- [ ] 保证金比例分级表是否及时更新
- [ ] 极端行情下保证金计算是否有压力测试

### 融资业务（P0）
- [ ] 融资额度控制是否严格（不超额放贷）
- [ ] 利息计算是否准确（精度、计息天数）
- [ ] 还款顺序是否正确（先息后本 or 先本后息）
- [ ] IPO 融资的保证金冻结和释放是否正确
- [ ] 融资标的名单更新是否及时

### 多币种（P1）
- [ ] 汇率更新是否及时
- [ ] 换汇操作是否原子性
- [ ] 汇率缓冲是否足够（防止汇率波动导致资金不足）
- [ ] 多币种总资产汇总计算是否正确

### 出入金（P1）
- [ ] 入金到账是否有防重机制
- [ ] 出金是否校验可提取金额
- [ ] 大额出入金是否有审核机制
- [ ] 反洗钱检查是否到位
