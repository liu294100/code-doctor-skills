参考富途/moomoo 等标杆产品，针对互联网券商多市场交易系统的深度审查与设计指导。

## 一、统一交易引擎架构

### 1.1 多市场统一委托模型

```
┌─────────────────────────────────────────────────────────┐
│                    统一委托网关                            │
│  协议适配 → 参数标准化 → 统一委托模型 → 路由分发             │
├─────────────────────────────────────────────────────────┤
│              市场路由层（按 market + secType 路由）          │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌───────────────┐ │
│  │ 港股引擎 │ │ 美股引擎 │ │ A股引擎  │ │ 期货/期权引擎  │ │
│  └────┬────┘ └────┬────┘ └────┬────┘ └──────┬────────┘ │
├───────┼──────────┼──────────┼───────────────┼──────────┤
│       ↓          ↓          ↓               ↓           │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌───────────────┐ │
│  │港交所网关│ │美股通道  │ │沪深交易所│ │ 期货交易所     │ │
│  │OCG/BSS  │ │FIX 4.2  │ │STEP/Bin │ │ CTP/飞马       │ │
│  └─────────┘ └─────────┘ └─────────┘ └───────────────┘ │
└─────────────────────────────────────────────────────────┘
```

**统一委托数据模型**
```java
// 核心委托对象 — 所有市场统一
class UnifiedOrder {
    String orderId;          // 全局唯一委托编号（Snowflake）
    String accountId;        // 交易账户ID
    String market;           // 市场：HK/US/SH/SZ/HKCC/SGX...
    String secType;          // 证券类型：STOCK/ETF/WARRANT/OPTION/FUTURE/BOND/FUND
    String symbol;           // 证券代码（统一编码：market.code，如 HK.00700）
    String side;             // 方向：BUY/SELL/SHORT/COVER（融券卖出/买券还券）
    String orderType;        // 委托类型（见下方枚举）
    BigDecimal price;        // 委托价格（市价单为0）
    BigDecimal quantity;     // 委托数量
    BigDecimal filledQty;    // 已成交数量
    BigDecimal avgPrice;     // 成交均价
    String timeInForce;      // 有效期：DAY/GTC/GTD/IOC/FOK
    String status;           // 状态（统一状态机）
    String currency;         // 币种：HKD/USD/CNY
    Long createTime;         // 委托时间（UTC毫秒）
    Long updateTime;         // 最后更新时间
    String exchangeOrderId;  // 交易所委托编号
    String errorCode;        // 废单原因码
    Map<String, String> extFields; // 市场特有扩展字段
}
```

**统一委托类型枚举**
```
通用类型：
  LIMIT          — 限价单（所有市场）
  MARKET         — 市价单（美股/期货）
  STOP           — 止损单
  STOP_LIMIT     — 止损限价单
  TRAILING_STOP  — 追踪止损单

港股特有：
  ENHANCED_LIMIT — 增强限价单（最多匹配10个价位）
  AUCTION        — 竞价单（集合竞价时段）
  AUCTION_LIMIT  — 竞价限价单
  SPECIAL_LIMIT  — 特别限价单（最多匹配1个价位）
  ODD_LOT        — 碎股单（不足一手）
  DARK_POOL      — 暗盘单（IPO暗盘交易）

美股特有：
  MARKET_ON_OPEN  — 开盘市价单（MOO）
  MARKET_ON_CLOSE — 收盘市价单（MOC）
  LIMIT_ON_OPEN   — 开盘限价单（LOO）
  LIMIT_ON_CLOSE  — 收盘限价单（LOC）
  EXTENDED_HOURS  — 盘前盘后限价单

期货/期权特有：
  FAK            — Fill And Kill（部分成交剩余撤销）
  FOK            — Fill Or Kill（全部成交或全部撤销）
```

### 1.2 统一委托状态机

```
                    ┌──→ 已废单（REJECTED）
                    │
待提交(PENDING) ──→ 已提交(SUBMITTED) ──→ 已确认(CONFIRMED)
                    │                        │
                    │                   ┌────┴────┐
                    │                   ↓         ↓
                    │            部分成交      全部成交
                    │         (PARTIAL_FILLED) (FILLED)
                    │                   │
                    │                   ↓
                    │              部分撤单
                    │           (PARTIAL_CANCELLED)
                    │
                    └──→ 待撤(CANCEL_PENDING) ──→ 已撤(CANCELLED)
                    └──→ 待改(MODIFY_PENDING) ──→ 已改(MODIFIED) ──→ 已确认
```

**状态流转规则**
- 只有 CONFIRMED / PARTIAL_FILLED 状态可以发起撤单
- 只有 CONFIRMED / PARTIAL_FILLED 状态可以发起改单
- FILLED / CANCELLED / REJECTED 是终态，不可变更
- 改单实现：港股支持原地改单，美股通常是撤旧下新（Cancel-Replace）
- 状态回退禁止：状态只能前进不能后退，防止并发导致状态错乱

## 二、港股交易专项

### 2.1 港交所交易时段

```
时段              时间              支持委托类型
─────────────────────────────────────────────────
开市前时段         09:00-09:15      竞价单/竞价限价单
（输入买卖盘）
对盘前时段         09:15-09:20      不可输入/修改/撤销
（随机对盘）
暗盘时段           09:20-09:30      不可输入（等待开盘）
持续交易（上午）    09:30-12:00      所有限价类型
午间休市           12:00-13:00      可输入委托（排队）
持续交易（下午）    13:00-16:00      所有限价类型
收市竞价时段        16:00-16:08      竞价单/竞价限价单
（随机收市 16:08-16:10）
```

### 2.2 港股交易规则引擎

**价格校验**
- 限价单：价格必须在 24 个价位（spread table）范围内
- 增强限价单：可偏离最佳价格最多 10 个价位
- 价位表（Spread Table）：不同价格区间有不同最小变动单位
  ```
  价格区间          最小变动价位
  0.01 - 0.25      0.001
  0.25 - 0.50      0.005
  0.50 - 10.00     0.010
  10.00 - 20.00    0.020
  20.00 - 100.00   0.050
  100.00 - 200.00  0.100
  200.00 - 500.00  0.200
  500.00 - 1000.00 0.500
  1000.00 - 2000.00 1.000
  2000.00 - 5000.00 2.000
  5000.00+          5.000
  ```
- 校验逻辑：委托价格必须是对应价位的整数倍

**数量校验**
- 整手交易：每只股票的每手股数不同（需维护证券信息表）
- 碎股交易：不足一手的只能通过碎股市场卖出，不能买入
- 碎股单独处理：碎股委托走专门的碎股通道

**港股通特殊处理**
- 可交易标的：仅限港股通标的名单内的股票
- 名单更新：每半年调整一次，需及时更新
- 额度控制：每日额度限制，实时查询剩余额度
- 假期安排：港股通在内地或香港任一方休市时均不交易
- 不支持暗盘交易、不支持碎股买入

### 2.3 港股 IPO 交易

**IPO 认购流程**
```
IPO 公告 → 认购期开放（通常 3-5 天）
         → 客户提交认购申请
           ├→ 现金认购：冻结认购资金 + 手续费
           └→ 融资认购：冻结保证金（通常 10%）+ 融资利息
         → 认购截止 → 汇总提交（通过中间人/CCASS）
         → 公布中签结果
           ├→ 中签：扣款 + 分配股份 → 上市首日可交易
           └→ 未中签：解冻资金
         → 暗盘交易（上市前一天 16:15-18:30）
         → 正式上市
```

**IPO 系统设计要点**
- 认购资金预冻结：提交认购时立即冻结，不等中签
- 融资认购：需要独立的 IPO 融资额度管理
- 超额认购倍数展示：实时统计认购倍数供客户参考
- 中签分配：按交易所规则分配（甲组/乙组/回拨机制）
- 暗盘交易：独立的暗盘撮合通道，非交易所撮合
- 一人一手保证：小额认购优先保证中签

### 2.4 牛熊证/窝轮交易

**牛熊证（CBBC）**
- 牛证（Bull）：看涨，标的价格上涨时获利
- 熊证（Bear）：看跌，标的价格下跌时获利
- 强制收回机制：标的价格触及收回价时，牛熊证立即停止交易并强制收回
  - 收回价 = 行使价（N类，无剩余价值）
  - 收回价 ≠ 行使价（R类，可能有剩余价值）
- 系统处理：
  - 实时监控标的价格与收回价的关系
  - 触发收回时：立即停止该牛熊证的交易，撤销所有未成交委托
  - 计算剩余价值（R类）并入账

**窝轮（Warrants）**
- 认购证（Call Warrant）：看涨
- 认沽证（Put Warrant）：看跌
- 到期处理：
  - 价内（In the Money）：自动行使，计算结算价值入账
  - 价外（Out of the Money）：自动作废，持仓清零
- 展示信息：引伸波幅、有效杠杆、溢价、到期日、行使价、换股比率

**系统设计**
- 证券信息维护：牛熊证/窝轮的收回价、行使价、到期日、换股比率等
- 到期日管理：定时任务扫描即将到期的衍生品，提前通知客户
- 收回事件处理：订阅交易所收回通知，实时处理

## 三、美股交易专项

### 3.1 美股交易时段

```
时段              时间（ET）         时间（北京）       说明
──────────────────────────────────────────────────────────
盘前交易           04:00-09:30       17:00-22:30*     仅限价单
正常交易           09:30-16:00       22:30-05:00*     所有订单类型
盘后交易           16:00-20:00       05:00-09:00*     仅限价单
* 夏令时提前1小时
```

### 3.2 美股交易规则引擎

**价格校验**
- 最小变动价位：$0.01（价格 >= $1）/ $0.0001（价格 < $1）
- 涨跌幅限制：无固定涨跌幅，但有 LULD 机制（Limit Up-Limit Down）
  - 价格偏离参考价超过一定比例时触发交易暂停
  - 暂停 5 分钟后恢复交易
- 熔断机制（Market-Wide Circuit Breaker）：
  - Level 1：S&P 500 下跌 7%，暂停 15 分钟
  - Level 2：S&P 500 下跌 13%，暂停 15 分钟
  - Level 3：S&P 500 下跌 20%，当日停止交易

**PDT 规则（Pattern Day Trader）**
- 适用条件：保证金账户（Margin Account）
- 规则：5 个交易日内日内交易（同日买卖同一证券）>= 4 次
- 账户要求：被标记为 PDT 后，账户净资产必须 >= $25,000
- 违规处理：低于 $25,000 时限制日内交易，只能平仓不能开新仓
- 系统实现：
  - 实时统计滚动 5 日内的日内交易次数
  - 下单前校验：如果本次交易会触发 PDT 且资产不足，提示警告
  - 提供 PDT 次数展示：已用 X/3 次（剩余可用次数）

**碎股交易（Fractional Shares）**
- 支持最小 0.0001 股交易
- 碎股不直接发送到交易所，由券商内部撮合或做市商处理
- 碎股订单类型：仅支持市价单
- 碎股持仓展示：精确到 4 位小数
- 公司行为处理：碎股分红按比例计算，精确到分

**美股期权交易**
- 期权链展示：按到期日 × 行权价矩阵展示 Call/Put
- 期权策略支持：
  - 单腿：买入/卖出 Call/Put
  - 多腿组合：Spread/Straddle/Strangle/Iron Condor/Butterfly
- 行权处理：
  - 美式期权：到期前任意时间可行权
  - 自动行权：到期日价内 $0.01 以上自动行权
  - 行权需要足够的资金（Call 行权）或持仓（Put 行权）
- 希腊字母展示：Delta/Gamma/Theta/Vega/Rho 实时计算

### 3.3 美股订单路由

```
客户委托 → 智能路由引擎（SOR）
          ├→ 交易所直连（NYSE/NASDAQ/ARCA/BATS...）
          ├→ 暗池（Dark Pool）
          └→ 做市商（Market Maker）
```

**智能订单路由（SOR）策略**
- 最佳执行（Best Execution）：选择最优价格的交易场所
- 流动性搜索：大单拆分到多个场所执行，减少市场冲击
- 成本优化：考虑各场所的交易费用和返佣（Maker-Taker 模型）
- 合规要求：SEC Rule 606 要求披露订单路由信息

## 四、A 股北向交易（沪深港通）

### 4.1 北向交易规则

**可交易标的**
- 沪股通：上证 180 + 上证 380 成分股 + 沪深 300 沪市成分股
- 深股通：深证成分指数 + 中小创新指数成分股 + 沪深 300 深市成分股
- 标的名单定期调整，需及时更新

**交易规则差异**
- T+0 回转交易：北向投资者买入 A 股当日可卖出（与内地 T+1 不同）
- 涨跌幅：主板 ±10%，科创板/创业板 ±20%，北交所 ±30%
- 最小单位：100 股（科创板/创业板/北交所可 1 股）
- 交易时段：跟随 A 股交易时段（9:15-15:00）
- 不支持融资融券、不支持大宗交易

**额度管理**
- 每日额度：沪股通/深股通各 520 亿人民币
- 额度实时查询：对接交易所额度接口
- 额度用尽处理：暂停买入，允许卖出
- 总额度：已取消

**汇率处理**
- 参考汇率：T 日开盘前公布，用于冻结资金
- 结算汇率：T+1 日公布，用于实际结算
- 汇率缓冲：买入时按参考汇率上浮 3% 冻结
- 汇率差退款：结算时多冻结的部分退回

### 4.2 北向交易系统设计

**前端交易所对接**
- 通过港交所 CSC（China Stock Connect）系统接入
- 协议：基于 OCG 协议扩展
- 报盘限制：遵守交易所流控规则

**持仓管理**
- 实益持有人模式：通过香港中央结算（HKSCC）代持
- 持仓查询：T 日成交后 T+1 日交收完成才正式入账
- 权益处理：分红、送股等通过 HKSCC 代为处理

## 五、条件单与智能订单

### 5.1 条件单引擎

```
┌─────────────────────────────────────────┐
│              条件单引擎                    │
│                                          │
│  行情订阅 ──→ 条件评估器（内存计算）        │
│               │                          │
│               ├→ 价格条件（>=/<=/突破）     │
│               ├→ 涨跌幅条件（日涨跌幅）     │
│               ├→ 成交量条件                │
│               ├→ 技术指标条件（MA/MACD）    │
│               └→ 时间条件（定时触发）        │
│               │                          │
│               ↓                          │
│          条件满足 → 生成普通委托 → 提交交易  │
└─────────────────────────────────────────┘
```

**条件单类型**
- 止损单：价格跌破某值时卖出
- 止盈单：价格涨到某值时卖出
- 追踪止损：价格从最高点回落 X% 时卖出
- 到价买入：价格跌到某值时买入
- 条件组合：多个条件 AND/OR 组合

**设计要点**
- 条件评估频率：每个行情 Tick 都要评估（不能用定时轮询）
- 条件持久化：条件单存储在 DB，服务重启后自动恢复
- 跨交易日：GTC（Good Till Cancel）条件单跨交易日有效
- 触发确认：条件触发后生成的委托如果被废单，需要通知客户
- 并发控制：同一账户的多个条件单可能同时触发，注意资金/持仓竞争

### 5.2 智能订单类型

**冰山单（Iceberg Order）**
- 大单拆分为多个小单，每次只显示一部分
- 参数：总数量、显示数量、价格、间隔时间
- 实现：本地维护，每个小单成交后自动提交下一个

**时间加权平均（TWAP）**
- 在指定时间段内均匀下单
- 参数：总数量、开始时间、结束时间、单次数量
- 实现：定时器按间隔提交子单

**成交量加权平均（VWAP）**
- 按市场成交量比例下单，减少市场冲击
- 参数：总数量、参与率（占市场成交量的比例）
- 实现：监控市场成交量，按比例提交子单

## 六、交易系统性能与可靠性

### 6.1 性能指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 委托提交延迟 | < 10ms（P99） | 从收到请求到发送到交易所 |
| 成交推送延迟 | < 50ms（P99） | 从收到回报到推送客户端 |
| 委托吞吐量 | > 5000 笔/秒 | 单市场峰值 |
| 条件单评估 | < 1ms/条 | 单条件评估延迟 |
| 系统可用性 | 99.99% | 交易时段 |

### 6.2 可靠性设计

**委托不丢不重**
- 委托持久化：写入 DB + WAL 后才返回客户端确认
- 幂等提交：客户端生成 clientOrderId，服务端去重
- 回报不丢：交易所回报按序号处理，缺失序号主动补全
- 回报不重：按 exchangeOrderId + execId 去重

**故障恢复**
- 交易引擎重启：从 WAL 恢复未完成委托状态
- 报盘网关重启：重连交易所 + 按序号补全缺失回报
- 数据库故障：委托写入本地 WAL，DB 恢复后回放

### 6.3 交易审查检查清单

- [ ] 委托状态机是否严格，是否存在非法状态流转
- [ ] 资金/持仓冻结是否与委托提交在同一事务
- [ ] 改单/撤单是否处理了并发场景（撤单和成交同时到达）
- [ ] 条件单触发是否有防重复触发机制
- [ ] 多市场交易规则是否配置化而非硬编码
- [ ] 价格/数量校验是否覆盖所有市场的规则
- [ ] 交易时段校验是否考虑了节假日和特殊交易日
- [ ] 碎股/零股处理是否正确
- [ ] 汇率处理是否有缓冲机制
- [ ] 报盘网关是否有流控和重连机制
- [ ] 成交回报是否有去重和补全机制
- [ ] 委托日志是否完整（全链路 traceId）
