## 一、期货交易规则知识库

### 1.1 香港期货（HKEX Futures）

```
主要品种：
  恒指期货（HSI）：恒生指数期货
  小型恒指期货（MHI）：合约价值为 HSI 的 1/5
  国企指数期货（HHI）：H 股指数期货
  小型国企指数期货（MCH）
  MSCI 中国 A50 期货

合约规格（恒指期货 HSI）：
  合约乘数：$50 HKD / 点
  最小变动价位：1 点 = $50 HKD
  合约月份：即月、下月、及之后两个季月
  交易时段：
    日盘：09:15-12:00, 13:00-16:30
    收市后交易（T+1 Session）：17:15-03:00（次日）
  最后交易日：合约月份倒数第二个交易日
  结算方式：现金结算
  保证金：交易所保证金 + 券商加收

交易规则：
  T+0 双向交易（可做多可做空）
  逐日盯市（Mark to Market）
  强制平仓：保证金不足时
  持仓限额：交易所规定的最大持仓数量
```

### 1.2 美国期货（CME/CBOT/NYMEX/COMEX）

```
主要品种：
  股指：ES（标普500迷你）、NQ（纳指迷你）、YM（道指迷你）
  外汇：6E（欧元）、6J（日元）、6B（英镑）
  能源：CL（原油）、NG（天然气）
  金属：GC（黄金）、SI（白银）、HG（铜）
  农产品：ZC（玉米）、ZS（大豆）、ZW（小麦）

合约规格（ES 标普500迷你）：
  合约乘数：$50 USD / 点
  最小变动价位：0.25 点 = $12.50 USD
  合约月份：3/6/9/12 月（季月）
  交易时段：几乎 23 小时（周日18:00-周五17:00 ET）
  结算方式：现金结算
  保证金：CME SPAN 保证金系统

交易规则：
  T+0 双向交易
  涨跌停限制：分级熔断（7%/13%/20%）
  持仓报告：大户持仓报告制度
```

### 1.3 期货交易系统设计要点

**保证金管理**
```
初始保证金：开仓时需要的保证金
维持保证金：持仓期间最低保证金要求（通常为初始的 70-80%）

保证金计算：
  单品种：持仓手数 × 合约乘数 × 最新价 × 保证金比例
  跨品种：SPAN 保证金系统（组合保证金，考虑对冲效应）

逐日盯市：
  每日结算价确定后，计算持仓盈亏
  盈利：增加可用资金
  亏损：减少可用资金
  可用资金 < 0：追加保证金通知
  可用资金 < -(维持保证金 - 初始保证金)：强制平仓
```

**移仓换月（Roll Over）**
```
临近到期的合约需要移仓到下一个合约月份：
  平仓当前月合约 → 开仓下一月合约
  
自动移仓提醒：
  到期前 5 个交易日开始提醒
  到期前 1 个交易日强制提醒
  最后交易日未平仓：按结算价强制平仓（现金结算）
                    或进入交割流程（实物交割）
```

**夜盘处理**
```
夜盘属于下一个交易日：
  例：周一夜盘 17:15 开始 → 属于周二交易日
  
系统设计：
  交易日概念 ≠ 自然日
  夜盘和日盘的持仓/资金连续
  日终清算在日盘收市后进行（不等夜盘）
  夜盘的盈亏计入下一交易日
```

## 二、期权交易规则知识库

### 2.1 港股期权（HKEX Stock Options）

```
标的：部分港股蓝筹股 + 恒指/国企指数
合约规格：
  合约单位：因标的而异（通常 1000 股/张 或 100 股/张）
  最小变动价位：$0.01 HKD
  到期月份：即月、下两个月、及之后三个季月
  行权方式：美式（股票期权）/ 欧式（指数期权）
  交割方式：实物交割（股票）/ 现金结算（指数）

交易时段：
  09:30-12:00, 13:00-16:00（跟随正股）

行权处理：
  美式期权：到期前任意交易日可行权
  到期日自动行权：价内期权自动行权（可申请放弃）
  行权后：T+2 交收标的股票
```

### 2.2 美股期权（US Options）

```
标的：几乎所有美股 + 指数（SPX/NDX/VIX）
合约规格：
  合约单位：100 股/张（标准）
  最小变动价位：$0.01（>$3）/ $0.05（<=$3）
  到期类型：
    月期权：每月第三个周五到期
    周期权（Weekly）：每周五到期
    日期权（0DTE）：当日到期
  行权方式：美式（股票期权）/ 欧式（指数期权）

期权策略：
  单腿：
    买入看涨（Long Call）
    买入看跌（Long Put）
    卖出看涨（Short Call）— 需保证金
    卖出看跌（Short Put）— 需保证金

  多腿组合：
    垂直价差（Vertical Spread）：同到期日不同行权价
    日历价差（Calendar Spread）：同行权价不同到期日
    跨式（Straddle）：同行权价同到期日的 Call + Put
    宽跨式（Strangle）：不同行权价同到期日的 Call + Put
    铁鹰（Iron Condor）：4 腿组合
    蝶式（Butterfly）：3 个行权价的组合

保证金计算（卖出期权）：
  Naked Call：max(20% × 标的市值 - 虚值金额, 10% × 标的市值) + 权利金
  Naked Put：max(20% × 标的市值 - 虚值金额, 10% × 行权价) + 权利金
  Spread：最大亏损金额
  Covered Call：无额外保证金（持有标的）
```

### 2.3 期权系统设计要点

**期权链展示**
```
到期日选择：[2024-01-19] [2024-01-26] [2024-02-16] ...
                    
行权价    Call买价  Call卖价  |  Put买价  Put卖价
─────────────────────────────────────────────
$140      $15.20   $15.40   |  $0.45    $0.50
$145      $10.50   $10.70   |  $0.80    $0.90    ← ATM
$150      $6.30    $6.50    |  $1.60    $1.75
$155      $3.20    $3.40    |  $3.50    $3.70
$160      $1.40    $1.55    |  $6.20    $6.40

标注：ITM（价内）高亮，ATM（平值）标记
```

**希腊字母实时计算**
```
Delta (Δ)：标的价格变动 $1 时期权价格变动
Gamma (Γ)：Delta 的变化率
Theta (Θ)：每天时间价值衰减
Vega (ν)：波动率变动 1% 时期权价格变动
Rho (ρ)：利率变动 1% 时期权价格变动

计算模型：
  欧式期权：Black-Scholes 模型
  美式期权：二叉树模型 / BAW 近似
  引伸波幅（IV）：由市场价格反推波动率（Newton-Raphson 迭代）

计算频率：
  行情变化时实时计算（对活跃合约）
  每 5 秒批量计算（对非活跃合约）
```

**期权行权处理**
```
行权流程：
  客户提交行权申请（或到期自动行权）
  → 校验：持仓充足、资金/持仓满足交割要求
  → 冻结期权持仓
  → 提交行权指令到交易所
  → 交易所确认
  → 交割处理：
     Call 行权：支付行权价 × 合约单位，获得标的股票
     Put 行权：交付标的股票，获得行权价 × 合约单位
  → 更新持仓和资金

自动行权规则：
  到期日收盘后，价内 >= $0.01 的期权自动行权
  客户可在到期日 17:30 前申请放弃行权
  资金/持仓不足时不自动行权（通知客户）

被行权（Assignment）：
  卖出期权的客户可能被行权
  随机分配（交易所随机选择被行权的卖方）
  被行权后需要交付标的或资金
```

**多腿期权组合下单**
```
组合下单流程：
  客户选择策略类型 → 选择各腿参数 → 计算组合保证金
  → 提交组合订单 → 交易所撮合（组合成交或不成交）

组合保证金优化：
  Spread：保证金 = 最大亏损（而非各腿保证金之和）
  Iron Condor：保证金 = max(Call Spread 最大亏损, Put Spread 最大亏损)
  
组合盈亏图：
  根据各腿参数计算到期日盈亏曲线
  展示最大盈利、最大亏损、盈亏平衡点
```

## 三、牛熊证与窝轮知识库

### 3.1 牛熊证（CBBC - Callable Bull/Bear Contracts）

```
产品结构：
  牛证（Bull）：看涨标的资产
  熊证（Bear）：看跌标的资产
  
关键参数：
  标的资产：恒指、个股等
  行使价（Strike Price）：到期结算的参考价
  收回价（Call Price）：触发强制收回的价格
  到期日：合约到期日期
  换股比率：每份牛熊证对应多少标的单位
  
分类：
  N 类（收回价 = 行使价）：收回后无剩余价值
  R 类（收回价 ≠ 行使价）：收回后可能有剩余价值

收回机制：
  牛证：标的价格 <= 收回价 → 强制收回
  熊证：标的价格 >= 收回价 → 强制收回
  收回后：
    N 类：直接作废，价值归零
    R 类：进入结算期（1个交易日），计算剩余价值
    R 类剩余价值 = (结算价 - 行使价) / 换股比率（牛证）
                  = (行使价 - 结算价) / 换股比率（熊证）

定价公式（简化）：
  牛证理论价 = (标的价格 - 行使价) / 换股比率 + 财务费用
  熊证理论价 = (行使价 - 标的价格) / 换股比率 + 财务费用

系统处理：
  1. 实时监控标的价格与收回价的关系
  2. 收回触发：
     → 立即停止该牛熊证交易
     → 撤销所有未成交委托
     → 冻结持仓
     → 计算剩余价值（R类）
     → 通知持有人
  3. 到期处理：
     → 价内：按结算价计算价值，现金入账
     → 价外：作废，持仓清零
```

### 3.2 窝轮（Warrants）

```
产品结构：
  认购证（Call Warrant）：看涨
  认沽证（Put Warrant）：看跌

关键参数：
  标的资产、行使价、到期日、换股比率
  引伸波幅（Implied Volatility）
  有效杠杆（Effective Gearing）
  溢价（Premium）
  到期前日数

定价：Black-Scholes 模型
  认购证价格 = BS_Call(S, K, T, r, σ) / 换股比率
  认沽证价格 = BS_Put(S, K, T, r, σ) / 换股比率

关键指标计算：
  Delta = ∂价格/∂标的价格 / 换股比率
  有效杠杆 = Delta × 标的价格 / (窝轮价格 × 换股比率)
  溢价(Call) = (行使价 + 窝轮价格 × 换股比率 - 标的价格) / 标的价格
  打和点(Call) = 行使价 + 窝轮价格 × 换股比率

到期处理：
  价内（ITM）：自动行使，计算结算价值
    Call 结算价值 = max(结算价 - 行使价, 0) / 换股比率
    Put 结算价值 = max(行使价 - 结算价, 0) / 换股比率
  价外（OTM）：自动作废

系统设计：
  - 窝轮/牛熊证信息每日更新（新上市/到期/收回）
  - 引伸波幅实时计算（反推 IV）
  - 到期日管理：提前通知客户即将到期的持仓
  - 收回事件实时处理
  - 展示排序：按成交量/引伸波幅/有效杠杆/到期日排序
```

## 四、衍生品技术方案模板补充

### 4.1 期货技术方案额外章节

**保证金系统设计**
- 保证金计算引擎（SPAN 或简化模型）
- 实时保证金监控
- 追保通知流程
- 强平引擎设计
- 保证金参数管理（交易所保证金 + 公司加收）

**结算系统设计**
- 逐日盯市结算流程
- 结算价获取和校验
- 盈亏计算和资金划转
- 交割处理（现金/实物）

**移仓换月**
- 到期提醒机制
- 自动移仓策略（可选）
- 移仓成本计算

### 4.2 期权技术方案额外章节

**定价引擎**
- 定价模型选择（BS/二叉树/蒙特卡洛）
- 希腊字母计算
- 引伸波幅曲面（Volatility Surface）
- 计算性能优化（缓存/预计算/GPU 加速）

**期权链管理**
- 合约生成规则（新到期日/新行权价）
- 合约信息维护
- 期权链展示优化

**组合保证金**
- 策略识别引擎
- 组合保证金计算
- 组合盈亏图生成

**行权/被行权处理**
- 行权申请流程
- 自动行权规则
- 被行权随机分配
- 交割处理

### 4.3 牛熊证/窝轮技术方案额外章节

**收回事件处理**
- 实时价格监控引擎
- 收回触发和通知
- 剩余价值计算
- 持仓清理

**到期处理**
- 到期日扫描
- 结算价值计算
- 自动入账/清零
- 客户通知

**产品筛选器**
- 多维度筛选（标的/类型/到期日/杠杆/溢价）
- 排序和比较
- 实时数据刷新

## 六、衍生品审查检查清单

- [ ] 保证金计算是否实时准确

- [ ] 强平触发和执行是否可靠

- [ ] 希腊字母计算是否准确
