## 一、基金交易知识库

### 1.1 基金类型与交易方式

```
按交易方式分类：
┌──────────────────────────────────────────────────────┐
│ 场内基金（Exchange-Traded）                            │
│  ETF：交易所实时交易，T+0（港股/美股）或 T+1（A股）      │
│  LOF：既可场内交易也可场外申赎                           │
│  交易方式：与股票相同，通过交易所委托买卖                  │
├──────────────────────────────────────────────────────┤
│ 场外基金（OTC Fund）                                   │
│  公募基金：开放式基金，通过基金公司/代销平台申购赎回        │
│  私募基金：合格投资者，最低投资门槛（通常100万）           │
│  交易方式：申购/赎回，按净值（NAV）成交                   │
└──────────────────────────────────────────────────────┘
```

### 1.2 场外基金交易规则

**申购（Subscribe）**
```
申购流程：
  客户提交申购 → 资金冻结 → T 日 15:00 前提交为 T 日申请
  → 基金公司确认 → T+1 公布净值 → 计算份额
  → T+2 份额到账（不同基金确认时间不同）

份额计算：
  申购份额 = (申购金额 - 申购手续费) / T 日基金净值
  申购手续费 = 申购金额 × 申购费率
  申购费率：通常 1.5%（互联网平台打一折 0.15%）

申购限制：
  最低申购金额（通常 10 元起）
  单日申购上限（部分基金限购）
  大额申购暂停（基金公司公告）
  新基金认购期 vs 开放申购期
```

**赎回（Redeem）**
```
赎回流程：
  客户提交赎回 → 份额冻结 → T 日 15:00 前提交为 T 日申请
  → 基金公司确认 → T+1 公布净值 → 计算赎回金额
  → T+3 至 T+7 资金到账（不同基金不同）

赎回金额计算：
  赎回金额 = 赎回份额 × T 日基金净值 - 赎回手续费
  赎回手续费 = 赎回金额 × 赎回费率
  赎回费率（按持有时间递减）：
    < 7 天：1.5%（惩罚性费率，鼓励长期持有）
    7 天 - 1 年：0.5%
    1 年 - 2 年：0.25%
    >= 2 年：0%

赎回限制：
  最低赎回份额
  巨额赎回（单日赎回 > 基金总份额 10%）：基金公司可延期或部分确认
  基金暂停赎回（极端市场情况）
```

**基金转换**
```
A 基金 → B 基金（同一基金公司内）
  优势：比赎回再申购更快，手续费可能更低
  流程：赎回 A 份额 + 申购 B 份额，一次操作完成
  费用：转换费 = max(A赎回费, 0) + max(B申购费 - A申购费, 0)
```

**分红处理**
```
分红方式：
  现金分红：分红金额直接入账
  红利再投资：分红金额自动申购基金份额（免申购费）
  客户可设置默认分红方式

分红计算：
  分红金额 = 持有份额 × 每份分红金额
  红利再投份额 = 分红金额 / 除息日净值
```

### 1.3 基金系统设计要点

**净值管理**
```
净值来源：基金公司/数据商每日推送
净值发布时间：T+1 日 18:00-22:00（不同基金不同）
净值校验：与多个数据源交叉验证
历史净值存储：用于计算收益率、最大回撤等

净值数据模型：
  fundCode     — 基金代码
  navDate      — 净值日期
  nav          — 单位净值
  accNav       — 累计净值（含分红）
  dailyReturn  — 日收益率
  publishTime  — 发布时间
```

**基金产品信息**
```
基础信息：基金代码、名称、类型、基金公司、基金经理、成立日期
费率信息：申购费率、赎回费率、管理费、托管费、销售服务费
交易规则：最低申购额、最低赎回份额、确认天数、到账天数
风险等级：R1-R5
业绩指标：近1月/3月/6月/1年/3年收益率、最大回撤、夏普比率
持仓信息：前十大持仓、行业分布、资产配置
```

**基金交易状态管理**
```
申购单状态：
  待提交 → 已提交 → 已确认（份额到账）/ 失败（退款）

赎回单状态：
  待提交 → 已提交 → 已确认（资金到账）/ 失败（份额解冻）

关键时间点：
  T 日 15:00：申购/赎回截止时间
  15:00 前提交：按 T 日净值
  15:00 后提交：按 T+1 日净值
  节假日不是交易日：顺延到下一交易日
```

## 二、月供股票（Stock Savings Plan）知识库

### 2.1 月供股票产品设计

```
产品概念：
  定期定额投资股票/ETF，类似基金定投
  每月固定日期、固定金额自动买入指定股票
  支持碎股，小额即可参与（如每月 500 HKD 起）

参考富途月供股票：
  支持市场：港股、美股
  最低月供金额：港股 500 HKD，美股 100 USD
  扣款日：每月固定日期（如每月 8 日）
  执行方式：扣款日以市价买入
  手续费：通常免佣金或低佣金
  碎股支持：支持买入碎股（如 0.5 股腾讯）
```

### 2.2 月供股票系统设计

**计划管理**
```java
class StockSavingsPlan {
    String planId;           // 计划ID
    String accountId;        // 账户ID
    String market;           // 市场
    String symbol;           // 股票代码
    BigDecimal monthlyAmount;// 每月投资金额
    Integer executeDay;      // 每月执行日（1-28）
    String currency;         // 币种
    String status;           // 状态：ACTIVE/PAUSED/TERMINATED
    LocalDate startDate;     // 开始日期
    LocalDate endDate;       // 结束日期（可选）
    LocalDate nextExecuteDate;// 下次执行日期
}
```

**执行流程**
```
定时任务（每日检查）
  → 筛选今日需要执行的计划
  → 校验：账户状态正常、资金充足、股票可交易
  → 资金扣款（冻结月供金额）
  → 生成买入委托（市价单 or 限价单）
    ├→ 港股：开盘后以增强限价单买入
    └→ 美股：开盘后以市价单买入
  → 成交确认
    ├→ 全部成交：记录买入份额和成本
    ├→ 部分成交：剩余金额退回
    └→ 未成交：全额退回，记录失败原因
  → 更新计划执行记录
  → 通知客户执行结果

异常处理：
  执行日非交易日：顺延到下一个交易日
  资金不足：跳过本月，通知客户
  股票停牌：跳过本月，通知客户
  连续 3 个月失败：自动暂停计划，通知客户
```

**月供收益计算**
```
总投入 = Σ(每次买入金额)
总市值 = Σ(每次买入股数) × 最新价
总收益 = 总市值 - 总投入
收益率 = 总收益 / 总投入 × 100%
年化收益率 = (1 + 收益率) ^ (365 / 持有天数) - 1

平均成本 = 总投入 / 总股数（自动实现成本平均法 DCA）

展示：
  投资概览：总投入、当前市值、累计收益、收益率
  执行记录：每月买入价格、数量、金额
  成本分析：平均成本 vs 当前价格
  收益曲线：按月展示累计收益变化
```

## 三、IPO 认购知识库

### 3.1 港股 IPO

**认购方式**
```
现金认购：
  全额资金冻结
  手续费：每笔 50-100 HKD
  中签后扣款，未中签退款

融资认购（孖展）：
  保证金比例：5%-10%
  融资利率：年化 1%-4%（视市场竞争）
  融资倍数：最高 10-20 倍
  融资天数：认购期 + 公布结果（通常 5-10 天）
  中签后：融资部分开始正式计息
  未中签：退还保证金，收取少量利息

认购档位（甲组/乙组）：
  甲组：认购金额 <= 500 万 HKD
  乙组：认购金额 > 500 万 HKD
  回拨机制：超额认购倍数高时，从乙组回拨到甲组
  一人一手：甲组优先保证每人至少中一手
```

**IPO 系统流程**
```
IPO 公告发布 → 系统录入 IPO 信息
  → 开放认购（通常 3-5 天）
    → 客户选择认购方式和金额
    → 现金认购：冻结全额资金 + 手续费
    → 融资认购：冻结保证金 + 预占融资额度
  → 认购截止（通常 12:00 noon）
    → 汇总认购数据
    → 通过中间人/CCASS 提交认购
  → 等待分配结果（1-2 个交易日）
  → 公布中签结果
    → 中签处理：
       现金：扣款（已冻结）
       融资：实际放款，开始计息
       股份分配：记录中签股数
    → 未中签处理：
       现金：解冻资金
       融资：释放额度，退还保证金
       收取手续费（部分券商不中签不收费）
  → 暗盘交易（上市前一天 16:15-18:30）
    → 融资客户暗盘卖出优先还款
  → 正式上市
    → 股份可正常交易
    → 融资客户：可选择持有或卖出还款
```

### 3.2 美股 IPO

```
认购方式：
  通常通过承销商分配，散户参与机会有限
  部分互联网券商提供 IPO 申购通道

流程：
  IPO 路演 → 询价 → 定价 → 分配 → 上市
  
散户参与：
  提交认购意向（非保证分配）
  冻结认购资金
  分配结果通知
  上市首日可交易

系统设计：
  与美股 IPO 分配平台对接
  认购意向管理
  资金冻结/解冻
  分配结果处理
```

### 3.3 IPO 系统设计要点

**IPO 产品管理**
```java
class IpoProduct {
    String ipoId;              // IPO 编号
    String market;             // 市场：HK/US
    String symbol;             // 股票代码（上市后）
    String companyName;        // 公司名称
    BigDecimal offerPrice;     // 发行价（区间或固定）
    BigDecimal offerPriceLow;  // 发行价下限
    BigDecimal offerPriceHigh; // 发行价上限
    Integer lotSize;           // 每手股数
    LocalDateTime subscribeStart; // 认购开始时间
    LocalDateTime subscribeEnd;   // 认购截止时间
    LocalDate allotmentDate;   // 公布分配日
    LocalDate listingDate;     // 上市日期
    LocalDate darkPoolDate;    // 暗盘交易日（港股）
    Boolean marginEnabled;     // 是否支持融资认购
    BigDecimal marginRate;     // 保证金比例
    BigDecimal maxLeverage;    // 最大融资倍数
    BigDecimal financingRate;  // 融资利率
    String status;             // 状态：UPCOMING/SUBSCRIBING/CLOSED/ALLOTTED/LISTED
}
```

**认购倍数实时统计**
```
实时展示：
  总认购金额 / 发行总额 = 超额认购倍数
  甲组认购倍数 / 乙组认购倍数
  本平台认购人数和金额

数据来源：
  本平台数据：实时统计
  全市场数据：综合券商公布数据估算
  更新频率：每 5 分钟
```

## 四、其他产品知识库

### 4.1 债券交易

```
债券类型：
  政府债券：美国国债（T-Bill/T-Note/T-Bond）、港府债券
  企业债券：投资级/高收益债
  可转债：可转换为股票的债券

交易方式：
  场内交易：交易所挂牌，与股票类似
  场外交易（OTC）：报价驱动，通过做市商

关键概念：
  面值（Par Value）：通常 $1000 或 $100
  票面利率（Coupon Rate）：年利息 / 面值
  到期收益率（YTM）：持有到期的年化收益率
  久期（Duration）：价格对利率变动的敏感度
  应计利息（Accrued Interest）：买入时需支付上次付息日到交易日的利息

系统设计：
  债券信息维护：面值、票面利率、付息频率、到期日、信用评级
  应计利息计算：按实际天数/360 或实际天数/365
  到期收益率计算：Newton-Raphson 迭代求解
  付息处理：到期自动入账利息
  到期处理：到期自动赎回面值
```

### 4.2 结构性产品

```
常见类型：
  ELN（Equity Linked Note）：股票挂钩票据
  DCI（Dual Currency Investment）：双币投资
  Accumulator/Decumulator：累计/递减期权

ELN 示例：
  挂钩标的：腾讯（00700.HK）
  投资期限：1 个月
  行使价：当前价格的 90%
  年化收益率：15%（如果到期时标的价格 > 行使价）
  到期处理：
    标的价格 > 行使价：返还本金 + 利息
    标的价格 <= 行使价：按行使价接货（获得股票）

系统设计：
  产品创建和管理
  到期监控和自动处理
  盈亏计算
  风险提示（适当性管理）
```

### 4.3 暗盘交易（Grey Market / Dark Pool）

```
港股 IPO 暗盘：
  时间：上市前一天 16:15-18:30
  参与者：已认购该 IPO 的客户
  交易方式：券商内部撮合（非交易所）
  委托类型：限价单
  交收：与 IPO 股份一起 T+2 交收

系统设计：
  独立的暗盘撮合引擎（价格优先、时间优先）
  暗盘行情推送（买卖盘口、最新成交）
  暗盘交易与正式持仓的关联
  融资客户暗盘卖出：优先偿还 IPO 融资
  暗盘成交记录和对账
```

### 4.4 股票拆合股/供股/公开发售

```
拆股（Stock Split）：
  例：1 拆 10，持有 100 股 → 1000 股，股价 /10
  系统处理：
    除权日自动调整持仓数量和成本价
    历史 K 线复权处理
    委托价格校验更新

合股（Reverse Split）：
  例：10 合 1，持有 1000 股 → 100 股，股价 ×10
  碎股处理：不足 1 股的部分现金补偿

供股（Rights Issue）：
  现有股东按比例获得认购新股的权利
  流程：
    除权日 → 获得供股权 → 供股权交易期
    → 客户选择行权（认购）或卖出供股权或放弃
    → 行权截止 → 新股上市
  系统处理：
    供股权自动分配
    供股权交易支持
    行权申请和资金冻结
    新股入账

公开发售（Open Offer）：
  类似供股，但供股权不可交易
  客户只能选择认购或放弃
```

## 六、各产品审查检查清单

### 基金
- [ ] 净值获取和校验是否可靠
- [ ] 15:00 截止时间判断是否准确（含节假日）
- [ ] 申购/赎回费率计算是否正确
- [ ] 巨额赎回处理是否完善
- [ ] 分红处理（现金/再投资）是否正确
- [ ] 基金转换费用计算是否正确
- [ ] 份额精度是否足够（通常小数点后 2-4 位）

### 月供股票
- [ ] 执行日非交易日的顺延逻辑是否正确
- [ ] 资金不足时的处理是否合理
- [ ] 碎股买入和持仓展示是否正确
- [ ] 收益计算（DCA 成本均摊）是否准确
- [ ] 计划暂停/恢复/终止逻辑是否完整
- [ ] 并发执行（多个计划同时扣款）是否安全

### IPO
- [ ] 认购资金冻结是否及时准确
